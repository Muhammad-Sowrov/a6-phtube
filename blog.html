Discuss the scope of var, let, and const.

In JavaScript, var, let, and const are used to declare variables, but they have different scoping rules and behavior.

Scope of 'var':
1. Var can not be used outside of the function.
2. If a 'var' declare outside of a function then it becomes a global function.

Scope of 'let':
1. 'let' are blocked scope. It means they are not accessible outside of that block.
2. 'let' variable can be reassign.

Scope of 'const':
1. 'const' variable are also blocked scope like 'let'.
2. 'const' variable are constant. That means they can not be reassign.

2. Tell us the use cases of null and undefined.

Undefined means the variable has been declared, but its value has not been assigned. Null means an empty value or a blank value.

Use cases of null:
1. When you don't have the value then you can use 'null' as absence of value.
2. You can use 'null' as temporary placeholder for a value that will be assigned later.

Use case of undefined:
1. When parameter is not declare then it will automatically set 'undefined'.
2. When function is called with fewer data that needed then missing data will set to as 'undefined' .
